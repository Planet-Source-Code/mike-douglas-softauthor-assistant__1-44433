Data Screen: Submit

Once data has been completely entered, this screen is used to aide in the creation of documents as well as submission of the information to email addresses and web sites.

Data on this screen is seperated for the two basic submission types, E-Mail and Web. The submission type is selected by the radio buttons in the upper right-hand portion of the screen.

Command Buttons
To the right of this screen are several command buttons enabling functions for this screen.
PAD Create PAD HTML and XML files based on current program information.
DIZ Create FILE_ID.DIZ file based on current program information.
ReadME Create README.TXT file based on current program information.
License Create LICENSE.TXT file based on current program information.
Press Create press release (PRESSRELEASE.TXT) file based on current program information.
HTML Create HTML file (for use as product home page) based on current program information.
ALL Create all above listed files based on current program information.
   
Auto Update Connect to the Aesgard Technologies web site and download updated data files.
SUBMIT Used to activate submission process for current submission type.

Data Fields
E-Mail Submission  
To A list of selected email address to send submission information to.
From Name The proper name (Ex: Joe Smith), used for replies to email submissions.
From E-Mail The email address (Ex: JoeSmith@domain.om), used for replies to email submissions.
Server Settings
Address TCP/IP address of your SMTP server. Can be proper (mail.domain.com) or numeric (111.111.111.111).
Port TCP/IP port of server used for SMTP traffic (usually 25).
Timeout Number of seconds to wait for server responses before halting request.
Retries Number of times to retry mail send before halting request.
SMTP Authentication
User Name If authentication is required to send mail through SMTP server, this is the User Name/Account used.
Password If authentication is required to send mail through SMTP server, this is the password used.
   
Web Submission  
Sites A list of selected web sites to send submission information to.
Do not submit to completed sites This will cause Web Submitter's browse buttons to skip over sites marked as completed.
Refresh Submit List  
URLs When checked this will enable the "Current Site" and "All Sites" buttons to clear URLs updated during previous submission cycles.
Completion Flags When checked this will enable the "Current Site" and "All Sites" buttons to clear completion flags set during previous submission cycles.
